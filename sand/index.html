Current version

Dev Sandbox IV is a departure from the previous Dev Sandbox versions. It is more powerful, but less cluttered. It is also closer to having a plugin system.
In Dev Sandbox IV, instead of having grouped categories, there is just one box set, and types of code are indicated by headers, like "css:", "html:", "jade:", and "js:". Put these at the beginning of the boxes and separate them from the rest of the code with a newline to indicate what type of code a box contains.
E. g. 
  css:
  body {
    font-family: "Comic Sans MS";
  }
  .annoying {
    color: #FFF;
    background: #F0F;
  }

  html:
  <h1>HANS'S DANK INTERSITEPAGE</h1>

  js:
  setInterval(() => document.querySelector("h1").classList.toggle("annoying"), 50);
In the future, I want to add a "jade-locals:" header that precedes some JSON to be passed to compiled Jade.

Version II: DSII

This little tool is an HTML data URI which contains a textbox and a button that says "GO!". The "go" button, when clicked, opens a new tab containing the code entered into the textbox. This works well offline and I'm hoping to add more things like HTML tag autoclosing, indentation, prettier UI, bookmarklet plugins for it, etc. It works offline as long as your entered HTML doesn't reqire online resources to function.
